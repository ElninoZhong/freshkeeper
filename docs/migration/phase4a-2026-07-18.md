## Hardening delivered

1. Skills update planning now fails closed:
   - missing lock skips without invoking an updater;
   - malformed lock reports failure and never broadens scope;
   - global `skills update -y` requires `FRESHKEEPER_ALLOW_GLOBAL_SKILLS_UPDATE=1`;
   - legacy `~/Documents/Claude` discovery and newest-cache execution were removed;
   - npx fallback is pinned to `skills@1.5.16`.
2. `enabledAdapters` now builds the runtime Registry; disabled adapters are not detected or updated, and unknown IDs fail visibly.
3. Scheduler writes validate single-line cron input, preserve unrelated entries, pass data through stdin to `crontab -`, distinguish no-crontab from read failure, and surface uninstall write failure.
4. The CLI awaits asynchronous actions with `parseAsync`; adapter failures set a non-zero process exit code.
5. Changelog fetch is best-effort with a timeout, so network failure cannot overturn an already completed update run.
6. The unused `simple-git` dependency was removed.
7. Durable `AGENTS.md`, `SOURCE_OF_TRUTH.md`, `CONTEXT.md`, and thin `CLAUDE.md` now define the project and safety seams.

## Deliberately deferred

- Project-lock refresh still does not enforce `computedHash` or a pinned revision and has no staged backup/atomic rollback. This needs a transactional design before unattended shared-library writes can be called fully protected.
- The `check` interface is still unsupported by production adapters and should either gain real implementations or stop presenting an all-clear result.
- The Codex adapter detects a standalone `codex` command but updates a Claude plugin; installation-source detection and duplicate-plugin updates need a separate product decision.
- Release tag/version/main-ancestry guards and the unused notification configuration remain follow-up work.

## Rollback

If path rollback is required, first verify the old path is still the compatibility symlink and the new worktree is clean. Remove only that symlink, atomically move the canonical directory back to the old location, and then update the registry. No data copy or Git history rewrite is required.
